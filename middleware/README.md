# Custom middleware

This example shows how to plug classic `net/http` middleware into a server
generated by `swagger generate server`.

Origin: questions from the go-swagger issue tracker, which definitely deserve an example:

- [#2683](https://github.com/go-swagger/go-swagger/issues/2683) — inject
  security response headers on every response.
- [#1120](https://github.com/go-swagger/go-swagger/issues/1120) — expose a
  `/metrics` endpoint scraped by Prometheus alongside the swagger API.

Both are wired through the `setupGlobalMiddleware` / `setupMiddlewares` hooks
that codegen already leaves in `restapi/configure_*.go`. **No `--exclude-main`
and no custom router needed** — that workaround suggested in #1120 is not
required.

## Spec

A minimal greeter API with two routes:

```yaml
paths:
  /greet:
    get: { ... }
  /greet/{name}:
    get: { ... }
```

The two distinct route templates make it easy to see that the Prometheus
`route` label uses the swagger template (`/api/greet/{name}`) rather than
the literal request path (`/api/greet/alice`).

## Generation

```sh
swagger generate server -A Greeter -f ./swagger.yml
```

## Where the middleware is wired

The generated `restapi/configure_greeter.go` exposes two hook points; this
example uses both:

```go
// setupGlobalMiddleware wraps EVERYTHING the server serves, including
// /swagger.json and the embedded UI. It runs before swagger routing.
func setupGlobalMiddleware(handler http.Handler) http.Handler {
    sec := secure.New(secure.Options{ /* ... */ })
    return metrics.Mount(sec.Handler(handler))
}

// setupMiddlewares wraps the swagger handler AFTER routing.
// middleware.MatchedRouteFrom is populated here.
func setupMiddlewares(handler http.Handler) http.Handler {
    return metrics.Instrument(handler)
}
```

Reading the chain from outermost to innermost:

1. `metrics.Mount` — intercepts `GET /metrics` so scrape traffic bypasses the
   swagger router and every middleware below.
2. `secure.Handler` — adds security response headers to every other response,
   including the spec document and the UI.
3. The swagger handler (routes, binding, validation, operation handler).
4. `metrics.Instrument` — runs once a route has been matched, captures the
   final status code and records the request count / duration with the
   swagger route template as a label.

## Security headers

Configured with [unrolled/secure](https://github.com/unrolled/secure):

| Option                       | Header                             | Value                                       |
|------------------------------|------------------------------------|---------------------------------------------|
| `STSSeconds`+`...Subdomains` | `Strict-Transport-Security`        | `max-age=63072000; includeSubDomains`       |
| `FrameDeny`                  | `X-Frame-Options`                  | `DENY`                                      |
| `ContentTypeNosniff`         | `X-Content-Type-Options`           | `nosniff`                                   |
| `ReferrerPolicy`             | `Referrer-Policy`                  | `no-referrer`                               |
| `CustomBrowserXssValue`      | `X-XSS-Protection`                 | `0` (modern guidance; see below)            |

`CustomBrowserXssValue: "0"` is deliberate. Modern OWASP / MDN guidance is to
**disable** the legacy XSS auditor outright, not enable it. Issue #2683 cites
`X-XSS-Protection: 1; mode=block` — that value is now considered harmful in
some browsers, which is why this example diverges from the issue text. The
default `BrowserXssFilter: true` would send the issue's value; using
`CustomBrowserXssValue` overrides it.

`ForceSTSHeader: true` is set so HSTS is visible even when curling over HTTP.
In production you would serve over HTTPS (the library detects TLS connections
automatically) or set `SSLProxyHeaders` behind a TLS-terminating proxy, and
leave `ForceSTSHeader` at the default `false`.

Content-Security-Policy is application-specific and is not enabled here. See
the [`unrolled/secure` README](https://github.com/unrolled/secure#options) for
the full option set (`ContentSecurityPolicy`, `PermissionsPolicy`,
`ExpectCTHeader`, …).

## Prometheus metrics

Exposed at `GET /metrics` using `promhttp.Handler()` on the default global
registry. Two metrics, both labelled `(method, route, code)`:

- `http_requests_total` — counter
- `http_request_duration_seconds` — histogram (default buckets)

The single piece of go-swagger-specific glue is the `route` label. It uses
`middleware.MatchedRouteFrom(r).PathPattern` so that
`GET /api/greet/alice` and `GET /api/greet/bob` both appear under
`route="/greet/{name}"`. This is essential for keeping Prometheus label
cardinality bounded; using the literal path would create one time series per
distinct URL.

Production setups often prefer a private `prometheus.NewRegistry()` so
multiple servers in the same process do not share metrics. That is out of
scope here — the example uses the default global registry to keep the focus
on middleware wiring.

## Run it

```sh
go run ./middleware/cmd/greeter-server --port 8080
```

Hit an endpoint:

```sh
curl -i http://127.0.0.1:8080/api/greet
```

```http
HTTP/1.1 200 OK
Content-Type: application/json
Referrer-Policy: no-referrer
Strict-Transport-Security: max-age=63072000; includeSubDomains
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-Xss-Protection: 0
Date: ...

{"message":"hello"}
```

Scrape metrics:

```sh
curl http://127.0.0.1:8080/metrics
```

```text
# HELP http_requests_total Total number of HTTP requests served, ...
# TYPE http_requests_total counter
http_requests_total{code="200",method="GET",route="/api/greet"} 1
http_requests_total{code="200",method="GET",route="/api/greet/{name}"} 3
# HELP http_request_duration_seconds Latency of HTTP requests, ...
# TYPE http_request_duration_seconds histogram
http_request_duration_seconds_bucket{code="200",method="GET",route="/api/greet",le="0.005"} 1
...
```

The route label includes the swagger `basePath` (`/api`) because the matched
route template is the full path served by the router.

A useful PromQL query for an API SLO dashboard:

```promql
rate(http_requests_total[1m])
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))
```

## Other middleware patterns, not shown here

Any standard `http.Handler` middleware composes the same way. A few examples
you may want to add to your own server using `setupGlobalMiddleware`:

- Panic recovery — `github.com/gorilla/handlers.RecoveryHandler`, or a small
  handwritten one (the generated comment explicitly suggests this).
- Request ID — `github.com/go-chi/chi/v5/middleware.RequestID` or similar.
- Access log — anything that wraps `http.Handler` and logs after the response.

## The two extension points

| Hook                     | Runs                                  | Sees matched route? | Use it for                                  |
|--------------------------|---------------------------------------|---------------------|---------------------------------------------|
| `setupGlobalMiddleware`  | Before swagger routing, wraps everything (spec, UI, all routes) | No | Cross-cutting concerns: security headers, panic recovery, metrics mount, global request ID. |
| `setupMiddlewares`       | After swagger routing, only for matched operations | Yes (`middleware.MatchedRouteFrom`) | Per-route concerns: instrumentation labelled by template, route-aware auth, …             |
